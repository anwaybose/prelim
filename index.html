Personalized Federated Learning

Personalized Federated Learning

  • aims to find such a global model \(\boldsymbol{\omega}^{(f)}_{mT}\) which doesn’t diverge from the global optima \(\boldsymbol{\omega}^{(c)}_{mT}\) also decreasing the distance between global update and the local update \(\boldsymbol{\omega}^{(k)}_{mT}\) after \(m\) finite synchronization, while the data distribution is non-IID, i.e., \(\mathbb{E}_{\mathcal{P}_k} [F_k(\boldsymbol{\omega} )] \neq f(\boldsymbol{\omega} )\).

\(\min_{\boldsymbol{\omega}} \{ \lVert \boldsymbol{\omega}^{(f)}_{mT} - \boldsymbol{\omega}^{(c)}_{mT} \rVert + \sum_{k=1}^K \lVert \boldsymbol{\omega}^{(f)}_{mT} - \boldsymbol{\omega}^{(k)}_{mT} \rVert \}\)

such that, \(\mathbb{E}_{\mathcal{P}_k} [F_k(\boldsymbol{\omega} )] \neq f(\boldsymbol{\omega} )\)

Personalized Federated Learning via Variational Bayesian Inference (pFedBayes)

Bayesian Neural Network (BNN)

  • BNN considers that each model parameter follows a certain distribution and aims to estimate the distribution parameters

\(p(y|\boldsymbol{X},\mathcal{D}) = \int p(y|\boldsymbol{X, \omega})p(\boldsymbol{\omega}|\mathcal{D})\)

Federated Bayesian Estimation (FedBE)

  • In a federated system clients are expected to learn the local model \(p(\boldsymbol{\omega}_k|\mathcal{D}_k)\) and the server intends to find the ensemble model

\(p(y|\boldsymbol{X},\mathcal{D}) \approx \frac{1}{N}\sum_{k=1}^N p(y|\boldsymbol{X, \omega}_k)p(\boldsymbol{\omega}_k|\mathcal{D}_k)\)

  • assumed that in FedBE \(\boldsymbol{\omega}_k \sim p(\boldsymbol{\omega}|\mathcal{D})\)

Personalized Federated Learning via Variational Bayesian Inference (pFedBayes)

BNN model via variational inference

  • each parameter (weight or bias) is characterized by a Gaussian distribution, \(\boldsymbol{\omega}_k \sim \mathcal{N}(\mu_k, \sigma_k^2)\)

  • The output of the DNN model is denoted by \(f_{\boldsymbol{\theta}}(\boldsymbol{X})\), where \(\boldsymbol{\theta} \in \mathbb{R}^T\) represents the vector of all model parameters

  • minimize the KL divergence between the estimate \(q(\boldsymbol{\theta})\) and the posterior distribution \(\pi(\boldsymbol{\theta}|\boldsymbol{\mathcal{D}})\)

\(\min_{q(\boldsymbol{\theta}) \in \mathcal{Q}} \quad KL(q(\boldsymbol{\theta}) || \pi(\boldsymbol{\theta}|\boldsymbol{\mathcal{D}}))\)

\(\pi(\boldsymbol{\theta}|\boldsymbol{\mathcal{D}}) \propto \pi(\boldsymbol{\theta}) p_{\boldsymbol{\theta}}(\mathcal{D})\) and \(\pi(\boldsymbol{\theta})\) denotes the prior and \(p_{\boldsymbol{\theta}}(\mathcal{D})\) denotes the likelihood.

\(\min_{q(\boldsymbol{\theta}) \in \mathcal{Q}} \quad -\mathbb{E}_{q(\boldsymbol{\theta})}[\log p_{\boldsymbol{\theta}}(\mathcal{D})] + KL(q(\boldsymbol{\theta}) || \pi(\boldsymbol{\theta}))\)

Personalized Federated Learning via Variational Bayesian Inference (pFedBayes)

pFedBayes

  • two step optimization process

  • \(\text{Server:} \quad \min_{\omega(\boldsymbol{\theta}) \in \mathcal{Q}_\omega} \quad \{ f(\boldsymbol{\omega}) = \frac{1}{N} \sum_{k=1}^N f_k(\boldsymbol{\omega})\}\)

  • \(\text{Clients:} \quad f_k(\boldsymbol{\omega}) = \min_{q_k(\boldsymbol{\theta}) \in \mathcal{Q}_k} \{ -\mathbb{E}_{q_k(\boldsymbol{\theta})}[\log p^k_{\boldsymbol{\theta}}(\mathcal{D}_k)] + \zeta KL(q_k(\boldsymbol{\theta}) || \omega(\boldsymbol{\theta})) \}\)

Gaussian Process Regression (GP)

  • BO typically employs a GP as a surrogate to model the objective function \(f\).
  • GP is characterized by its mean \(\mu(x)\) and a kernel function \(\kappa(x,x')\) for all \(x,x' \in \mathcal{X}\)
  • At the \(t+1\) iteration, the GP posterior \(\mathcal{GP}(\mu_t, \sigma_t^2)\) is represented by

\(\mu_t(\boldsymbol{x}) =\boldsymbol{k}_t( \boldsymbol{x})^T(\boldsymbol{K}_t - \lambda \boldsymbol{I})^{-1} \boldsymbol{y}_t, \quad \sigma_t^2(\boldsymbol{x, x'}) = \kappa(\boldsymbol{x, x'}) \boldsymbol{k}_t( \boldsymbol{x})^T(\boldsymbol{K}_t - \lambda \boldsymbol{I})^{-1} \boldsymbol{k}_t( \boldsymbol{x'})\)

where \(\boldsymbol{k}_t( \boldsymbol{x}) = \kappa(\boldsymbol{x, x_{t'}})\), \(\boldsymbol{K}_t = \kappa(\boldsymbol{x_{t'}, x_{t''}})\)

Differentially Private Federated Bayesian Optimization with Distributed Exploration

Federated BO

  • Random Fourier Feature (RFF) to approximate the kernel function \(\kappa\), \(\phi: \kappa \approx \phi(\boldsymbol{x})^T \phi(\boldsymbol{x'})\) \(\sup \vert \kappa(\boldsymbol{x}, \boldsymbol{x}') - \phi(\boldsymbol{x})^T \phi(\boldsymbol{x'})\rvert < \epsilon\)

  • \(\boldsymbol{\Phi_t} = (\phi(\boldsymbol{x}_{t'}))^T\), \(\boldsymbol{\Sigma}_t = \boldsymbol{\Phi_t}^T \boldsymbol{\Phi_t} - \lambda \boldsymbol{I}\) and \(\boldsymbol{\nu}_t = \Sigma_t^{-1} \Phi_t^T \boldsymbol{y}_t\).

\[\boldsymbol{\omega} \sim \boldsymbol{\mathcal{N}}(\boldsymbol{\nu}_t, \lambda \boldsymbol{\Sigma}_t^{-1})\]

  • set \(\tilde{f}(\boldsymbol{x}) = \phi(\boldsymbol{x})^T \boldsymbol{\omega}\)

  • selects \(\boldsymbol{x}_t = \text{arg} \max \boldsymbol{\phi}(\boldsymbol{x})^T \boldsymbol{\omega}_{k,t}\)

  • client then queries \(\boldsymbol{x}_t\) to observe \(\boldsymbol{y}_t\) and updates the GP posterior belief with \((\boldsymbol{x}_t, \boldsymbol{y}_t)\)

Future Work

Future Work

  • BO has a big application in localization and mapping. It’ll be interesting to work on secure, distributed localization and mapping using DP-FTS-DE

  • \(\boldsymbol{\omega}_t \sim \boldsymbol{\mathcal{N}}(\boldsymbol{\nu}_t, \lambda \boldsymbol{\Sigma}_t^{-1}) + \varepsilon\), where \(\varepsilon\) modelled from \(KL(\boldsymbol{\mathcal{N}}(\boldsymbol{\nu}_t^k, \lambda \boldsymbol{\Sigma}_{t,k}^{-1}) || \boldsymbol{\mathcal{N}}(\boldsymbol{\nu}_t, \lambda \boldsymbol{\Sigma}_t^{-1}))\)

  • Federated BO considers each client is modelling their own balck-box function. So, potentially federated learning can be applied on different clients learning different different models but on same features.

  • applying homomorphic encryption to the local model updates in pFedME and investigate if this approach has the potential to preserve convexity and smoothness while concealing the models from prying servers.
